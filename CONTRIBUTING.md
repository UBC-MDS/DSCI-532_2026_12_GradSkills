# Contributing

Contributions are welcome for this package.

There are many ways to contibute including:

* [Report bugs](#report-bugs)
* [Write Documentation](#write-documentation)
* [Submit Feedback](#submit-feedback)

## Report Bugs

Report bugs at [issues](https://github.com/UBC-MDS/DSCI-532_2026_12_GradSkills/issues).

If you are reporting a bug, please follow the template guidelines. The more
detailed your report, the easier and thus faster we can help you.

## Write Documentation

We are always appreciative of additional documentation for the graduate skills
dashboard. Whether that is as part of the official documentation,
in docstrings, or in other forums, just
[open an issue](https://github.ubc.ca/mds-2025-26/DSCI_532_viz-2_students/issues)
to share what you want to work on and we can help provide any additional
information you might need.

## Submit Feedback

The best way to send feedback is to file an issue through
[issues](https://github.ubc.ca/mds-2025-26/DSCI_532_viz-2_students/issues).
If your feedback fits the format of one of the issue templates, please use that.
Please remember that this package was developed as part of the UBC Master of Data Science
program in partial fulfillment of the requirements for DSCI 532 Data Vizualization 2.
As such, we may not be able to respond immediately.

## Getting Started

If you would like to contribute to the project, read below. Please make sure you
reach out to the team to discussion your idea before working on it and submitting
a pull request.

### Clone Repository, Create Branch, Code, and Submit Pull Request

1. Clone this [repository](https://github.com/UBC-MDS/DSCI-532_2026_12_GradSkills) and activate the environment.

```bash
git clone git@github.com:UBC-MDS/DSCI-532_2026_12_GradSkills.git
```

Navigate to the root directory of the project, before running the following code.

```bash
conda env create -n graduate_skills -f environment.yml
conda activate graduate_skills
```

2. Create a branch that will contain your commits.

3. Continually commit your changes and push your branch to GitHub. Make commit messages clear.

```bash
    git add . # or select appropriate files
    git commit -m "fix: summarize your changes"
    git push -u origin <branch_name>
```

4. Once your suggestions are ready to be reviewed, submit a pull request.
